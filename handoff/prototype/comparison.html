§ 3 — Vantaggi concreti

1
L'anno diventa protagonista — hai detto che era un pain point

Oggi è un badge piccolo nell'angolo. In A+ è il secondo elemento dell'header, in serif italic 32px. Appena arrivi sulla mappa capisci quando stai guardando. Costo: zero funzionale — sposta un elemento esistente.

2
Header leggibile — da 7 bottoni a 4 link

Embed + OpenAPI spostati in un menu "More" o nel footer API. Ask Claude resta come unica CTA. Il selettore lingua diventa un link testuale. Risultato: -50% di rumore visivo, stessa funzionalità raggiungibile.

3
Sidebar più leggera — filtri collassati

Era + Status aperti di default (i più usati), Region + Overlay chiusi. La lista "On map" resta sempre visibile in fondo come risultato live del filtro. Meno scroll, meno overwhelm.

4
Timeline come prodotto, non come utility

60px con i tick delle ere sopra il cursore. Quando trascini, vedi in tempo reale in che era stai entrando. L'era attiva è in Playfair italic — lo stesso linguaggio del badge anno, quindi il sistema visivo è coerente.

5
Confini cartografici, non "geometrie dati"

Stroke più sottile (0.9px vs 1px), fill tenue (28% vs 25%), palette HSL per entità invece di rosso/blu fisso. Risultato: sembra un atlante storico, non un dashboard. Comunica autorevolezza scientifica — il tuo posizionamento "machines first, honest with humans".

6
Coerenza visiva con homepage (dopo fase 2)

Gli stessi token (ambra #e8b14a, Playfair per hero numbers) applicati a landing + app. Un umano passa da homepage a mappa senza sentire un salto tonale. Brand unificato.

7
Effetto wow per umani, invisibile per agenti

Le API REST + MCP + la struttura OpenAPI non cambiano. Gli agenti AI continuano a vedere esattamente gli stessi endpoint. Solo gli umani che aprono /app vedono la differenza. Costo agente: zero. Costo umano: migliore ritenzione.

Verdetto

Cosa cambia: UI layer (CSS + qualche ritocco HTML). Tipografia, palette, layout sidebar/header/timeline.

Cosa non cambia: API, DB, GeoJSON, logica Leaflet, dati degli imperi, backend, MCP server, struttura URL, comportamento per gli agenti AI.

Stima: 1-2 giorni di lavoro Claude Code su un branch isolato, deploy controllato tramite il tuo cra-deploy.sh atlaspi.
